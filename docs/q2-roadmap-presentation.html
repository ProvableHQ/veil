PHASE 1: FIGURE IT OUT

Build the veil Prototype

What We're Doing

  • Build a working prototype in the veil repo
  • Mirror viem's patterns: Clients, Transports, Actions
  • Every action ships with agent tool + MCP tool
  • Zero hard dependencies on any specific SDK
  • Interface-first: wallets and SDKs plug in

What We Learn

  • Which viem patterns map cleanly to Aleo
  • Where Aleo needs its own abstractions (records, proving, ZK)
  • How agents interact with privacy-first chains
  • What the right package boundaries are

PHASE 3: BUILD IT OUT

Integrate With Aleo Tool Ecosystem

ProvableKit Integration

Concrete implementations of the veil interfaces backed by ProvableKit. If you import the SDK, you get real signing, proving, and key management.

Wallet + SDK + Mobile

Concrete veil adapters for the wallet standard, @provablehq/sdk, and Shield Mobile. Swap the adapter, everything else stays the same.

Aleo Hooks (aleo-wagmi)

Dogfood veil by building React hooks: useBalance, useReadContract, useWriteContract. Developers get wagmi-level ergonomics for Aleo dapps.

Leo Program Testing

Set veil as the standard JS/TS interface as the JS/TS test harness for Leo programs. Developers use the same API to test and to ship.
